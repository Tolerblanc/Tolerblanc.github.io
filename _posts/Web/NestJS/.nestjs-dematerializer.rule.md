# NestJS 해체분석기 Principle

**공식 문서 + 직접 경험의 결합**
- NestJS 공식 문서를 토대로, 직접 다뤄본 경험과 노하우를 적절히 버무려 독자들에게 ‘내 것으로 만든 지식’을 전달합니다.
- 단순히 공식 문서 내용을 요약하기보다, “왜 이런 구조인지?”, “설계 의도가 무엇인지?”, “어떤 상황에서 유리한지?” 등 해부학적 시선으로 접근합니다.

**기술적 원리 중심 + 실용 예제 병행**
- “NestJS는 이런 원리로 동작한다”라는 해체분석(내부 구조) 파트를 먼저 다룬 뒤,
- 실제 현업에서 어떻게 쓰면 좋은지(실무 예제, 응용 방법)를 이어서 보여주는 방식을 추천합니다.
- 독자들이 “아, 그래서 이렇게 동작하는 거구나”를 느낀 뒤, “실제 코드에선 이렇게 적용하면 좋다”라는 흐름으로 이어가면 이해가 잘 됩니다.

**한 게시글, 한 핵심 주제**
- 한 번에 너무 광범위하게 다루기보다는, DI, 라이프사이클, 모듈 시스템, 미들웨어, 필터, 인터셉터, 파이프, 가드, 마이크로서비스 등 NestJS의 주요 구조와 기능을 하나씩 쪼개서 집중 탐구하는 편이 좋습니다.
- 글이 길어질 수 있으니, “핵심 개념 → 내부 구조 → 사용 방법 → 주의사항과 팁”의 4단계로 구분 지어 정리해두면 가독성이 올라갑니다.

**코드 예제 + 실습 가이드 제공**
- 글에 첨부하는 예제 코드는 간단하면서도 핵심을 보여줄 수 있어야 합니다.
- 단순 예시(Hello World)에서 끝나지 않고, 예를 들어 DI 글에서는 “커스텀 프로바이더”나 “동적 모듈” 같은 고급 주제까지 확장하면 글의 밀도가 높아집니다.
- 필요하다면, GitHub 등 저장소를 마련해 실제 동작하는 샘플 프로젝트를 공유할 수도 있습니다.

**비교 & 대조**
- “NestJS가 일반 Express와 무엇이 다른가?”, “NestJS에서 A 라는 기능은 Spring의 어떤 부분을 연상시키는가?”처럼 타 프레임워크나 익숙한 개념과 비교하면 이해도가 빨라집니다.
- “NestJS 미들웨어 vs Express 미들웨어”, “NestJS CLI vs Angular CLI” 등, 독자들이 ‘아, 이런 차이가 있구나’를 체감하면 좋습니다.

**정리 & 마무리**
- 각 포스트 끝에는 “핵심 요약”과 “더 살펴볼 거리” 같은 섹션을 배치해 글 전체의 핵심 포인트와 추가 학습 리소스(공식 문서의 특정 챕터 링크 등)를 짧게 안내합니다.
- 글 간 연결성을 위해, 다음 글에서 어떤 내용을 다룰지 예고하는 것도 좋습니다.

**NestJS 개요 & 철학**
- 왜 NestJS가 생겼고, 어떤 문제를 해결하는가?
- NestJS가 지향하는 객체 지향/함수형/FP 등 다양한 패러다임 결합 설명.
1. **모듈과 의존성 주입(DI) 기초**
   - AppModule 예시.
   - `@Module`, `@Injectable` 동작 구조, 프로바이더 등록 원리.
2. **프로바이더 스코프와 라이프사이클**
   - 싱글톤, 요청 스코프, 트랜지언트 스코프.
   - `OnModuleInit`, `OnApplicationBootstrap` 등 훅(Hook) 개념.
3. **컨트롤러와 라우팅**
   - `@Controller`, 데코레이터, HTTP 메서드 핸들러 원리.
   - 기존 Express 라우터와 비교.
4. **미들웨어, 가드, 인터셉터, 필터**
   - 파이프까지 포함해 AOP(관점 지향 프로그래밍) 구조 정리.
   - 각각의 계층에서 어떤 역할을 하는지 단계별 다이어그램.
5. **유효성 검사 & 파이프**
   - `class-validator`, `class-transformer` 와 NestJS 파이프 연동.
   - 내부 동작 방식 정리.
6. **커스텀 데코레이터 설계**
   - NestJS의 메타프로그래밍 기법, ReflectMetadata 사용.
7. **동적 모듈(Dynamic Module)**
   - 특정 조건(환경 변수, 설정)에 따라 모듈을 동적으로 세팅하는 방법.
   - `forRoot`, `forRootAsync` 패턴 등.
8. **테스팅 전략**
   - e2e 테스트와 단위 테스트 차이점.
   - DI 이용하여 목(Mock) 주입하는 방법.
9. **확장성: Microservices, GraphQL, WebSockets**
   - NestJS로 확장된 아키텍처를 구성할 때 각 패키지가 어떻게 동작하고 무엇이 다른지.
   - 메시지 브로커와 NestJS 마이크로서비스의 구조.
10. **실전 예제**
    - JWT를 이용한 인증/인가, DB 연동(ORM 사용), 로깅, 배포 시 고려사항 등.
    - “NestJS 해체분석기 시리즈”를 통해 배운 내용 총정리
